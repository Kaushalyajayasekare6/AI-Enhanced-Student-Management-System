# Timetable System - Complete Documentation

## Overview
The **Timetable System** manages the scheduling of classes for different grades and terms. It handles:
- Automatic generation of timetables based on grade level
- Assignment of subjects and teachers to time slots
- Storage of timetable information in the database
- Display of timetables for teachers, students, and administrators

---

## System Architecture

```
Frontend (React)
      ↓
TimetableModal Component (Assign teachers/subjects)
      ↓
Backend API Routes (/timetable)
      ↓
Timetable Controller (Business Logic)
      ↓
Timetable Model (MongoDB Schema)
      ↓
Database
```

## Database Schema (Timetable.js)

### Main Timetable Document
```javascript
{
  grade: Number (1-13),
  section: String (A, B, C, D),
  stream: String (null for grades 1-11, Science/Commerce/Technology for 12-13),
  term: Number (1, 2, or 3),
  year: Number (2024, 2025, etc),
  slots: [TimetableSlot],
  autoGenerated: Boolean,
  timestamps: { createdAt, updatedAt }
}
```

### TimetableSlot Structure
```javascript
{
  day: String (Monday, Tuesday, Wednesday, Thursday, Friday),
  timeSlot: String (e.g., "07:30-08:20"),
  subject: String (e.g., "English", "Maths", "Science"),
  teacherId: ObjectId (Reference to Teacher document),
  teacherStatus: String ("assigned" or "unassigned"),
  room: String (e.g., "Room 6A")
}
```

### Key Constraints
```javascript
// Prevents duplicate timetables for same class/term/year
unique: { grade: 1, section: 1, stream: 1, term: 1, year: 1 }
```

## API Endpoints (Routes)

### 1. **Generate Timetable for Single Class**
```
POST /api/timetable/generate
Body: {
  grade: 6,
  section: "A",
  term: 1,
  year: 2024
}
Response: Generated Timetable document with slots
```

### 2. **Generate Grade 1-5 Timetable**
```
POST /api/timetable/generate-grade1-5
Body: {
  grade: 3,
  section: "B",
  term: 1,
  year: 2024
}
Response: Timetable with 6 periods/day, 5 days
```

### 3. **Auto-Generate All Grade 1-5 Timetables**
```
POST /api/timetable/generate-grade1-5-all
Generates timetables for:
- All grades 1-5
- All sections (A, B, C, D)
- All terms (1, 2, 3)
- Current year
Response: Array of generated timetables
```

### 4. **Generate Grade 6-9 Timetable**
```
POST /api/timetable/generate-grade6-9
Body: {
  grade: 7,
  section: "C",
  term: 2,
  year: 2024
}
Response: Timetable with 8 periods/day, intelligent teacher assignment
```

### 5. **Auto-Generate All Grade 6-9 Timetables**
```
POST /api/timetable/generate-grade6-9-all
Generates timetables for:
- All grades 6-9
- All sections (A, B, C, D)
- All terms (1, 2, 3)
- Current year
Response: Array of generated timetables
```

### 6. **Get Timetable for a Class**
```
GET /api/timetable/class?grade=6&section=A&term=1&year=2024
Response: Timetable document with all slots and teacher info
```

### 7. **Get Teacher's Timetable**
```
GET /api/timetable/teacher
(Requires authentication)
Response: All timetable slots for logged-in teacher
```

### 8. **Get Student's Timetable**
```
GET /api/timetable/student
(Requires authentication)
Response: Timetable for student's grade and section
```

### 9. **Get All Timetables**
```
GET /api/timetable/all
(Admin only)
Response: Array of all timetables in the system
```

### 10. **Update Timetable**
```
PUT /api/timetable
Body: {
  timetableId: "...",
  slots: [updated slots],
  ...other updates
}
Response: Updated timetable
```

### 11. **Delete Timetable**
```
DELETE /api/timetable?id=...
(Admin only)
Response: Success/Error message
```

### 12. **Get Timetable Suggestions**
```
POST /api/timetable/suggestions
Body: {
  grade: 6,
  section: "A"
}
Response: Intelligent suggestions for teacher-subject assignments
```

- **Save Button**: Confirms the assignment
- **Cancel Button**: Closes the modal without saving

## Timetable Generation Algorithm

### For Grades 1-5
1. Get configured time slots (6 per day)
2. Get subjects for the grade
3. Distribute subjects across 5 days using round-robin
4. Create timetable slot for each day × period combination
5. Keep teacher assignment as unassigned (manual later)

### For Grades 6-9
1. Determine elective subject based on section (Art, Music, Dancing, Drama)
2. Create subject cycle: Core subjects + section elective
3. Distribute subjects across 8 periods × 5 days using round-robin
4. **Intelligent Teacher Assignment**:
   - Get all teachers in the school
   - For each slot, find teachers matching the subject
   - Use special matching logic for Religion, Civic Education, PTS, electives
   - Assign teacher with lowest current load (balancing)
5. Mark slots as "assigned" or "unassigned" based on teacher availability

### Special Teacher Matching Rules

```javascript
function getTeacherCandidates(subject, teachers) {
  switch(subject) {
    case "Religion":
      return teachers with subjects: [Buddhism, Hinduism, Islam, Catholicism, Christianity]
    case "Civic Education":
      return teachers with subjects: [Citizenship, Civilization, Social Science, Life Competencies]
    case "PTS (Physical Training)":
      return teachers with subjects: [Technology, Physical Education]
    case "Art":
      return teachers with subjects: [Art, ART]
    default:
      return teachers with exact subject match
  }
}
```

## Important Features

### 1. **Auto-Generation with Intelligent Assignment**
- Automatically distributes subjects across days
- Intelligently assigns available teachers
- Balances teacher workload

### 2. **Grade-Specific Configuration**
- Different time slots for different grades
- Different sets of subjects per grade
- Special handling for streams (Grades 12-13)

### 3. **Teacher Load Balancing**
- Tracks how many periods each teacher teaches
- Assigns teachers with lighter loads first
- Prevents overloading individual teachers

### 4. **Subject Aliases**
- Religion: Multiple religion subjects → Single "Religion" slot
- Civic Education: Multiple civic subjects → Single slot
- Electives: Section-specific subjects

### 5. **Multi-View Access**
- **Admin**: Can see all timetables, generate, edit
- **Teacher**: Can see own timetable + class they teach
- **Student**: Can see own class timetable

### 6. **Term-Based**
- Timetables differ by school term (1, 2, or 3)
- Each term can have different schedules

## Performance Considerations

### Database Indexing
```javascript
Index: { grade: 1, section: 1, stream: 1, term: 1, year: 1 }
Purpose: Fast lookup of specific class timetables
```

### Teacher Load Calculation
- For each timetable generation, system scans all existing slots
- Counts periods for each teacher
- Chooses teachers with lowest load

This can be optimized with a "teacher_loads" collection for real-time tracking.
